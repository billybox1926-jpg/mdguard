# Contributing

Thanks for your interest in improving `mdguard`.

## Development expectations

- Keep changes focused and reviewable.
- Include tests for new behavior.
- Prefer small, composable modules over large files.
- Preserve conservative autofix behavior.

## Workflow

1. Open an issue (or reference an existing one).
2. Create a branch for your change.
3. Add tests and documentation updates.
4. Ensure CI passes.
5. Open a pull request with rationale and examples.

## Code of conduct

Be respectful and constructive in discussions and reviews.
